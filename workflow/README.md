# cirro-jbrowse-config Nextflow Workflow

A minimal DSL2 Nextflow wrapper around the `cirro-jbrowse-config` Python CLI.
The workflow runs two processes in sequence:

1. **SELECT_FILES** — calls `cirro-jbrowse-config select --non-interactive` to build `inputs.json` from explicit track and FASTA parameters.
2. **GENERATE_AND_UPLOAD** — calls `cirro-jbrowse-config upload` to generate a JBrowse2 static site and publish it as a new Cirro dataset.

---

## Prerequisites

- [Nextflow](https://www.nextflow.io/docs/latest/install.html) >= 23.04
- `cirro-jbrowse-config` installed and available on `PATH` (or set `process.container` in `nextflow.config`)
- Cirro credentials configured (`cirro configure` or environment variables accepted by the Cirro SDK)

---

## Running the workflow

```bash
nextflow run /path/to/workflow/main.nf \
    --project_id        "proj-abc123" \
    --dataset_id        "ds-xyz789" \
    --assembly          "hg38" \
    --tracks            "bam:Sample1:alignments/sample1.bam" \
    --tracks            "bigwig:Coverage:coverage/sample1.bw" \
    --fasta             "proj-ref123:ds-ref456:ref/hg38.fa.gz" \
    --upload_project_id "proj-results999" \
    --dataset_name      "My JBrowse Browser" \
    --description       "Generated by Nextflow"
```

Pass `--tracks` once per track. Omit `--fasta` when no reference FASTA is needed.

### Using a container

Add a container image to `nextflow.config` (or pass it on the command line):

```bash
nextflow run /path/to/workflow/main.nf \
    -with-docker \
    --project_id ... \
    # (remaining params)
```

```groovy
// nextflow.config
process {
    container = 'quay.io/cirrobio/cirro-jbrowse-config:latest'
}
```

## Parameters

| Parameter          | Default                                              | Description                                                  |
|--------------------|------------------------------------------------------|--------------------------------------------------------------|
| `project_id`       | `null` (required)                                    | Cirro project ID containing input files                      |
| `dataset_id`       | `null` (required)                                    | Cirro dataset ID containing input files                      |
| `assembly`         | `hg38`                                               | Assembly name                                                |
| `tracks`           | `[]`                                                 | One or more `TYPE:NAME:FILE_PATH[:INDEX_PATH]` strings       |
| `fasta`            | `null`                                               | Optional reference FASTA: `PROJ_ID:DS_ID:FILE_PATH`         |
| `upload_project_id`| `null` (required)                                    | Cirro project ID to publish results into                     |
| `dataset_name`     | `JBrowse Config`                                     | Name for the uploaded Cirro dataset                          |
| `description`      | `Generated by cirro-jbrowse-config Nextflow workflow`| Description for the uploaded Cirro dataset                   |
| `output_dir`       | `jbrowse-site`                                       | Local directory for intermediate static site assets          |

---

## Outputs

| File               | Process              | Contents                                          |
|--------------------|----------------------|---------------------------------------------------|
| `inputs.json`      | SELECT_FILES         | Track and assembly spec consumed by the generator |
| `upload_result.txt`| GENERATE_AND_UPLOAD  | stdout from the upload command (includes dataset ID) |
